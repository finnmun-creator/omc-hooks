# OMC Hooks — 한국어 가이드

> [English README](./README.md)

## Claude Code, 이렇게 쓰면 불편하지 않으셨나요?

Claude Code는 강력하지만, 터미널에서 쓰다 보면 이런 불편함이 있습니다:

**매번 y/n을 눌러야 합니다.**
파일 하나 수정하려 해도, 명령어 하나 실행하려 해도, 터미널에 `Allow? (y/n)` 이 뜹니다. 파일 10개를 고치는 작업이면 y를 10번 눌러야 합니다.

**뭘 하려는 건지 파악이 어렵습니다.**
터미널에 텍스트가 쭉 나오는데, 지금 Claude가 하려는 게 안전한 건지, 위험한 건지 한눈에 보이지 않습니다. `rm -rf`같은 위험한 명령도 `ls`같은 안전한 명령도 똑같은 모습으로 물어봅니다.

**작업 모드를 매번 설명해야 합니다.**
"병렬로 빠르게 처리해줘", "깊이 분석해줘" 같은 작업 방식을 매번 길게 설명해야 합니다.

**세션이 끊기면 맥락이 사라집니다.**
Claude Code를 껐다 켜면 이전에 뭘 하고 있었는지, 어떤 모드였는지 다 잊어버립니다.

**한마디로:**
> Claude Code에 "눈"과 "기억"이 없습니다.

---

## omc-hooks는 뭘 해주나요?

omc-hooks는 Claude Code에 **7개의 훅**을 설치하는 확장 도구입니다. 단순히 팝업 하나 붙이는 게 아니라, 권한 관리부터 작업 자동화, 메모리, 세션 복원까지 Claude Code의 전반적인 사용 경험을 개선합니다.

비유하자면:

| 기존 Claude Code | omc-hooks 설치 후 |
|------------------|-------------------|
| 현관문에 초인종만 있는 집 | CCTV + 컬러 경고등 + 스마트홈 시스템이 달린 집 |
| 누가 오든 "열어줄까요?" 한마디 | 택배기사면 자동 오픈, 낯선 사람이면 빨간불 |
| 집 나갔다 오면 뭘 하고 있었는지 모름 | 나갔다 와도 하던 일 이어서 진행 |
| 매번 "빠르게 해줘"를 직접 타이핑 | 키워드 한마디에 모드 자동 전환 |

설치되는 7개 훅:

| 훅 | 한줄 설명 |
|----|----------|
| 권한 승인 팝업 | 위험도별 색상 GUI로 도구 실행 허용/거부 |
| 키워드 자동 감지 | 메시지에서 모드 키워드를 읽고 자동 활성화 |
| 질문 알림 팝업 | Claude가 질문하면 팝업으로 알려줌 |
| 위임 경고 | 소스 파일 직접 수정 시 서브에이전트 위임 권고 |
| 메모리 저장 | 중요 발견을 디스크에 기록해서 컨텍스트 압축에도 보존 |
| 세션 복원 | 이전 세션의 모드와 미완료 작업을 자동 복원 |
| 상태 표시줄 | 현재 모드, 토큰 사용량 등 실시간 HUD |

## 주요 기능 상세

### 1. 권한 승인 팝업 (Permission GUI)

Claude Code가 파일 수정, 명령어 실행 등을 하기 전에 GUI 팝업 창이 뜹니다.

**4단계 위험 분류**

| 단계 | 색상 | 예시 | 동작 |
|------|------|------|------|
| Tier 0 | — | `ls`, `git status`, 파일 읽기 | 팝업 없이 자동 통과 |
| Tier 1 | 초록 | `git commit`, `npm install`, `.md` 수정 | 초록색 팝업 (허용 기본) |
| Tier 2 | 노랑 | 소스 코드 수정, `curl`, 파이프 명령 | 노란색 팝업 (주의) |
| Tier 3 | 빨강 | `rm`, `sudo`, `.env` 수정 | 빨간색 팝업 (거부 기본) |

**4단계 승인 범위** — 팝업에서 선택할 수 있습니다:

| 범위 | 지속 | 설명 |
|------|------|------|
| 이번만 | 1회 | 딱 이 한 번만 허용 |
| 세션 | 24시간 | 이 작업 세션 동안 같은 패턴 자동 허용 |
| 항상 | 영구 | 이 패턴은 앞으로 계속 자동 허용 |
| 도구 전체 | 영구 | 이 도구의 모든 호출을 항상 허용 |

**추가 기능:**
- **버스트 모드** (Windows): 체크박스 하나로 5초간 연속 호출 자동 승인 — 파일 10개 수정할 때 유용
- **Diff 미리보기**: Edit 작업 시 삭제되는 코드(빨강)와 추가되는 코드(초록) 색상 표시
- **맥락 표시**: 내가 원래 뭘 요청했는지, Claude가 왜 이 도구를 쓰려는지 팝업에 함께 표시
- **키보드 단축키**: `[1]`~`[4]` 범위 선택, `Enter` 허용, `Esc` 거부

### 2. 키워드 자동 감지 (Keyword Detector)

메시지를 보낼 때마다 자동으로 키워드를 스캔하고, 해당하는 작업 모드를 Claude에게 주입합니다.

**작동 방식:**
사용자가 메시지를 보내면(UserPromptSubmit) → 키워드 스캔 → 매칭되면 모드 활성화 컨텍스트를 Claude에게 자동 주입

**감지하는 키워드:**

| 키워드 | 활성화 모드 | 효과 |
|--------|-----------|------|
| `ultrawork`, `ulw`, `uw` | Ultrawork 모드 | 병렬 실행, 백그라운드 에이전트 동시 가동, 완료까지 멈추지 않음 |
| `search`, `find in codebase`, `where is` | 검색 모드 | 백그라운드 탐색 에이전트를 자동 실행 |
| `analyze`, `investigate`, `debug` | 분석 모드 | 심층 코드 분석 컨텍스트 주입 |
| `think`, `ultrathink` | 확장 추론 모드 | 깊이 생각하는 모드 활성화 |
| `매출`, `마진`, `퍼널`, `코호트`, `roi` 등 | DA/PM 모드 | 데이터 분석 전문가 모드 활성화 |

**예시:**
- `"ulw 이 버그 고쳐줘"` → ultrawork 모드가 자동 켜지고, 병렬로 빠르게 처리
- `"이 코드 analyze해줘"` → 분석 모드가 켜지고, 탐색 에이전트가 코드를 깊이 조사
- `"매출 데이터 뽑아줘"` → DA/PM 모드가 켜지고, 데이터 분석에 최적화된 지시가 주입

**우선순위 인터럽트 (`^^`):**
작업 중간에 급한 일이 생겼을 때, `^^` 접두사로 현재 작업을 일시 중단할 수 있습니다.
- `^^ 이 에러 먼저 봐줘` → 진행 중인 ultrawork를 일시 중단하고 에러 확인 먼저 실행
- `^^끝` 또는 `^^done` → 인터럽트 종료

**부가 효과:**
모든 메시지의 첫 200자를 임시 파일에 저장해서, 권한 승인 팝업에 "원래 요청 내용"으로 표시합니다. 팝업에서 "이게 뭘 하려는 건지" 맥락을 볼 수 있는 이유가 이것입니다.

### 3. 질문 알림 팝업 (Input Notify)

Claude가 질문(`AskUserQuestion`)을 할 때, 팝업으로 알려줍니다.

**왜 필요한가?**
Claude가 백그라운드에서 작업 중일 때 질문을 하면, 터미널을 안 보고 있으면 놓칩니다. 이 훅은 팝업을 띄워서 "Claude가 질문하고 있어요!"라고 알려줍니다.

**간단한 질문** (선택지 2~4개):
- 파란색 팝업에 질문 내용과 번호 붙은 버튼이 표시됩니다
- `[1] 옵션A`, `[2] 옵션B` 형태로, 클릭하면 VS Code/터미널로 전환
- 키보드 단축키: `1`~`4`로 선택 가능
- 60초 후 자동 닫힘

**복잡한 질문** (복수 선택, 마크다운 포함):
- "터미널에서 응답이 필요합니다" 안내와 함께 "VS Code로 이동" 버튼 표시

**핵심 설계:**
팝업은 알림 목적이라 Claude를 멈추지 않습니다. 팝업을 띄우는 동시에 Claude는 계속 진행합니다.

---

### 4. 위임 경고 (Pre-Tool-Use)

Claude가 소스 파일을 직접 수정하려 할 때, 서브에이전트에게 위임하라고 권고합니다.

**왜 필요한가?**
oh-my-claudecode 아키텍처에서는 메인 Claude(지휘자)가 직접 코드를 수정하지 않고, executor 서브에이전트에게 위임하는 것을 권장합니다. 이 훅은 그 규칙을 부드럽게 상기시켜줍니다.

**동작:**
- Claude가 `.ts`, `.py`, `.go`, `.rs`, `.java` 등 소스 파일에 Edit/Write를 시도하면:
  → `[DELEGATION NOTICE]` 경고 메시지가 Claude 컨텍스트에 주입됨
  → "executor 에이전트에게 위임하세요"라는 권고 표시
- `.claude/`, `.omc/`, `CLAUDE.md` 등 설정 파일은 경고 없이 통과

**소프트 경고입니다** — 실제로 수정을 막지는 않습니다. Claude에게 "이건 위임하는 게 좋겠어"라고 알려주는 역할입니다.

**우선순위 인터럽트 연동:**
`^^` 인터럽트가 활성화된 동안 관련 없는 도구 호출을 최대 3회까지 차단합니다. 3회 이후에는 안전 밸브가 열려서 통과됩니다.

---

### 5. 메모리 저장 (Post-Tool-Use)

서브에이전트(Task)가 작업을 마친 후, 중요한 발견 사항을 디스크에 기록합니다.

**왜 필요한가?**
Claude Code는 대화가 길어지면 이전 내용을 압축(compaction)합니다. 중요한 발견이 압축 과정에서 사라질 수 있는데, 이 훅은 그것을 디스크 파일(`.omc/notepad.md`)에 저장해서 보존합니다.

**작동 방식:**
서브에이전트의 출력에서 `<remember>` 태그를 감지하면 자동으로 notepad.md에 기록합니다.

**두 가지 태그:**

| 태그 | 저장 위치 | 용도 |
|------|----------|------|
| `<remember priority>내용</remember>` | Priority Context 섹션 | 항상 로드되는 핵심 정보 (500자 이내) |
| `<remember>내용</remember>` | Working Memory 섹션 | 타임스탬프 포함 세션 메모, 7일 후 자동 정리 |

**notepad.md 구조:**
```markdown
# Notepad
## Priority Context    ← 핵심 발견 (항상 로드)
## Working Memory      ← 세션 메모 (자동 정리)
## MANUAL              ← 사용자 직접 작성 (절대 자동 수정 안 됨)
```

### 6. 세션 복원 (Session Start)

Claude Code를 새로 시작할 때, 이전 세션의 상태를 자동으로 복원합니다.

**세 가지를 확인합니다:**

**1) Ultrawork 모드 복원**
이전 세션에서 ultrawork 모드로 작업 중이었다면, 새 세션에서도 자동으로 ultrawork 모드가 활성화됩니다.
→ Claude에게 "이전 세션에서 ultrawork 작업이 진행 중이었습니다. 원래 요청: ..." 이라는 컨텍스트가 주입됩니다.

**2) 미완료 작업 감지**
이전 세션에서 완료하지 못한 태스크가 있으면 알려줍니다.
→ "이전 세션에서 N개의 미완료 작업이 있습니다. 계속 진행하세요."

**3) 인터럽트 정리**
이전 세션이 `^^` 인터럽트 도중에 비정상 종료되었으면, 상태를 깨끗하게 정리하고 중단되었던 모드를 복원합니다.

**핵심 가치:**
Claude Code를 껐다 켜도 작업 맥락이 유지됩니다. 긴 작업 도중에 세션이 끊겨도 이어서 할 수 있습니다.

---

### 7. 상태 표시줄 HUD (Status Line)

Claude Code 하단 상태 바에 실시간 정보를 표시합니다.

**표시 내용:**
- 현재 활성 모드 (ultrawork, analyze 등)
- 실행 중인 에이전트 수
- 토큰 사용량

oh-my-claudecode 메인 패키지의 HUD 모듈을 로드하는 래퍼이며, 메인 패키지가 설치되어 있어야 정상 작동합니다.

## 설치 방법

### 필요한 것

시작하기 전에 이 두 가지가 설치되어 있어야 합니다:

1. **Claude Code** — Anthropic의 CLI 도구 ([공식 문서](https://docs.anthropic.com/en/docs/claude-code))
2. **Node.js 18 이상** — [nodejs.org](https://nodejs.org)에서 다운로드

### 방법 1: npx로 설치 (가장 간단)

터미널에 이 한 줄만 입력하면 됩니다:

```
npx omc-hooks
```

끝입니다. Windows, macOS, Linux 모두 지원합니다.

**업데이트:**
```
npx omc-hooks@latest
```

**제거:**
```
npx omc-hooks uninstall
```

### 방법 2: GitHub에서 직접 받기

좀 더 직접 관리하고 싶다면:

**Windows:**
```powershell
git clone https://github.com/finnmun-creator/omc-hooks.git
cd omc-hooks
.\install.ps1
```

**macOS / Linux:**
```bash
git clone https://github.com/finnmun-creator/omc-hooks.git
cd omc-hooks
bash install.sh
```

### 설치하면 어디에 뭐가 생기나요?

| 설치되는 파일 | 위치 | 역할 |
|--------------|------|------|
| 훅 스크립트 6개 | `~/.claude/hooks/` | 핵심 동작 엔진 |
| Windows UI | `~/.claude/hooks/ui/win/` | 팝업 창 (Windows) |
| macOS UI | `~/.claude/hooks/ui/mac/` | 팝업 창 (macOS) |
| 상태 표시줄 | `~/.claude/hud/` | 토큰 HUD |
| 승인 규칙 | `~/.claude/gui-approvals.json` | 기본 승인 설정 |

기존 `settings.json`은 안전하게 병합됩니다. 이미 있는 훅 설정은 그대로 유지되고, omc-hooks 항목만 추가됩니다.

## 설치 후 어떻게 달라지나요?

### 시나리오 1: 평소처럼 사용 — 별도 설정 불필요

설치 후 Claude Code를 그냥 실행하면 됩니다. 7개 훅이 자동으로 동작합니다.

```
claude
```

### 시나리오 2: 안전한 명령 — 자동 통과

Claude가 `ls`, `git status`, 파일 읽기 등을 실행하려 할 때:
→ 팝업 없이 **자동으로 승인**됩니다. 작업 흐름이 끊기지 않습니다.

### 시나리오 3: 일반 작업 — 초록색 팝업

Claude가 `git commit`이나 `.md` 파일 수정을 하려 할 때:
→ **초록색 팝업**이 뜹니다. 내가 원래 뭘 요청했는지, Claude가 왜 이걸 하려는지 맥락이 함께 표시됩니다.
→ Enter를 누르면 승인. 범위를 선택해서 반복 승인도 가능합니다.

### 시나리오 4: 소스 코드 수정 — 노란색 팝업 + 위임 권고

Claude가 `.ts`, `.py` 등 소스 코드를 수정하려 할 때:
→ **노란색 팝업**이 뜹니다. 변경될 코드의 전후 비교(diff)가 색상으로 표시됩니다.
→ 동시에 Claude 컨텍스트에 "executor에게 위임하세요"라는 권고가 주입됩니다.

### 시나리오 5: 위험한 명령 — 빨간색 팝업

Claude가 `rm`, `sudo`, `.env` 수정 등을 시도할 때:
→ **빨간색 팝업**이 뜹니다. 기본값이 "거부"라서 실수로 승인할 위험이 줄어듭니다.
→ Tier 3에서는 "항상 허용"을 선택할 수 없습니다. 세션 단위까지만 가능합니다.

### 시나리오 6: 연속 수정 — 버스트 모드 (Windows)

Claude가 파일 10개를 연속으로 수정해야 할 때:
→ 첫 번째 팝업에서 "Burst Mode" 체크박스를 켜고 승인하면,
→ 5초간 같은 유형의 후속 호출이 **자동으로 승인**됩니다.

### 시나리오 7: 키워드로 모드 전환

`"ulw 이 버그 전부 고쳐줘"` 라고 입력하면:
→ 키워드 감지기가 `ulw`를 인식 → ultrawork 모드 활성화
→ Claude가 "ULTRAWORK MODE ENABLED!" 라고 응답하며 병렬 작업 시작
→ 별도 설정이나 긴 프롬프트 필요 없이 키워드 한 단어로 모드 전환

### 시나리오 8: Claude가 질문할 때

Claude가 "어떤 방식을 쓸까요?" 같은 질문을 하면:
→ **파란색 팝업**이 뜹니다. 선택지가 번호 버튼으로 표시됩니다.
→ 다른 창에서 작업 중이어도 팝업으로 알 수 있습니다.
→ 버튼을 누르면 VS Code/터미널로 자동 전환됩니다.

### 시나리오 9: 세션을 껐다 다시 켤 때

ultrawork 모드로 큰 작업 중에 Claude Code를 종료했다가 다시 시작하면:
→ "이전 세션에서 ultrawork 작업이 진행 중이었습니다" 메시지가 자동 주입
→ 미완료 태스크가 있으면 "N개의 미완료 작업이 있습니다"도 함께 알려줌
→ 이어서 작업 가능

### 시나리오 10: 작업 중 급한 일이 생겼을 때

ultrawork로 대규모 리팩토링 중인데 급한 버그가 발견되면:
→ `^^ 이 에러 먼저 봐줘` 입력
→ 진행 중인 작업이 일시 중단되고, 에러 확인을 먼저 실행
→ `^^끝` 입력하면 인터럽트 종료

## 설정 커스터마이즈

`~/.claude/gui-approvals.json`에서 동작을 조정할 수 있습니다:

```json
{
  "rules": [],
  "version": 2,
  "config": {
    "autoApproveTier0": true,
    "autoApproveTier1": false,
    "dangerousRequireDoubleConfirm": true,
    "burstWindowMs": 5000,
    "sessionApprovalTTLHours": 24
  }
}
```

| 옵션 | 기본값 | 설명 |
|------|--------|------|
| `autoApproveTier0` | `true` | Tier 0(안전) 명령 자동 승인 |
| `autoApproveTier1` | `false` | Tier 1(일반) 명령도 자동 승인할지 여부 |
| `dangerousRequireDoubleConfirm` | `true` | Tier 3(위험) 명령에 이중 확인 요구 |
| `burstWindowMs` | `5000` | 버스트 모드 지속 시간 (밀리초) |
| `sessionApprovalTTLHours` | `24` | 세션 승인의 유효 시간 |

## 훅 간 연동 구조

omc-hooks의 7개 훅은 독립적으로도 동작하지만, 서로 데이터를 주고받으며 연동됩니다:

```
[사용자 메시지 입력]
    │
    ├→ 키워드 감지기: 모드 키워드 스캔 + 사용자 맥락 저장
    │       │
    │       ├→ 권한 팝업: 저장된 맥락을 "원래 요청" 패널에 표시
    │       └→ 세션 복원: ultrawork 상태 파일을 읽어서 다음 세션에 복원
    │
    ├→ 위임 경고: 소스 파일 수정 시 executor 위임 권고
    │
    └→ 질문 알림: AskUserQuestion 시 팝업 표시

[도구 실행 완료 후]
    │
    └→ 메모리 저장: <remember> 태그를 notepad.md에 기록

[새 세션 시작 시]
    │
    └→ 세션 복원: 모드, 미완료 작업, 인터럽트 상태 복원
```

## 플랫폼별 참고사항

### Windows
- Windows Forms 기반의 풍부한 UI (색상 diff, 키보드 단축키)

- diff 색상 표시 미지원
